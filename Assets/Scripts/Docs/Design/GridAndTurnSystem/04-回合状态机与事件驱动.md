# 回合状态机与事件驱动

## S（场景）

回合制游戏里，UI 要在回合开始时显示"Day X · Marigold 回合"，单位要重置行动状态，AI 要在轮到它时自动开始思考，战斗动画播完前不能结束回合。这些子系统分散且解耦，不能互相直接调用。

## T（挑战）

如何让这些高度解耦的子系统协调运作，而不需要形成密集的相互引用？

## A（方案）

`TurnManager` 是唯一的"报时器"，通过 static 事件广播当前回合上下文：

```csharp
public enum TurnPhase { Start, Main, End }

public struct TurnContext {
    public int Day;
    public int PlayerIndex;
    public UnitFaction Faction;
    public TurnPhase Phase;
}

// 关键事件
public static event Action<int> OnDayChanged;
public static event Action<TurnContext> OnTurnStarted;
public static event Action<TurnContext> OnTurnMainPhase;
public static event Action<TurnContext> OnTurnEnded;
public static event Action<TurnContext> OnTurnIntroAnimationComplete;
```

回合流程：Start → Main → End → 推进下一玩家/天 → 循环。TurnManager 只管推进状态和广播事件，不操作具体单位。

生命周期逻辑委托给 `TurnLifecycleService`：回合开始时收取建筑收入、重置工厂状态。`PlayerTurnController` 通过 TargetingSession 管理攻击/补给/装载等交互状态，收到 UI 事件后路由到对应命令执行。

## R（收益）

- 强解耦：UI、AI、表现层各自只订阅关心的事件，不引用 TurnManager 以外的任何类
- 扩展性强：新增回合阶段只需加枚举值和对应事件，无需修改订阅方
- 动画同步：开场动画通过 ReportIntroAnimationComplete 回传，保证动画期间输入锁定
- 命令可回退：TargetingSession 管理攻击/补给等交互状态，ESC 取消时能正确回退行动标记
