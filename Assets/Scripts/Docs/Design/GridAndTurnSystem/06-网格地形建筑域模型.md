# 网格/地形/建筑基础域模型

## S（场景）

战棋世界的核心模型需要同时服务四个需求：游戏逻辑（某格能否通行）、渲染（3D 预制体缩放对齐）、AI 模拟（纯数据快照）、编辑器（场景可视化编辑）。这些需求混在一起会让对象变得臃肿。

## T（挑战）

如何让核心域模型足够稳定，同时满足逻辑、渲染、AI、编辑器四种不同视角的需求？

## A（方案）

分层建模，数据与实体分离：

**数据层**（纯 C# / ScriptableObject）：
- `TileBaseType` / `TilePlaceableType`：地形配置 SO，含预制体引用、防御星级
- `BuildingData`：建筑配置 SO，含收入、占领参数、阵营外观
- `BuildingState`：纯 C# 建筑状态，用于 AI 快照，无 Unity 依赖

**实体层**（MonoBehaviour）：
- `Cell`：单格域对象，持有 `BaseType` + `PlaceableType` + 当前单位引用，负责预制体适配（缩放/旋转/轴心对齐）和事件广播（OnUnitEntered）
- `MapRoot`：世界空间索引，`Dictionary<Vector2Int, Cell>` 实现 O(1) 格子查询
- `BuildingController`：建筑逻辑（占领、收入），持有 `BuildingData` + `BuildingState`

双层地形：每个格子有 Base（地基层）和 Placeable（放置层），Placeable 的防御星级覆盖 Base。桥梁例外——无论在哪层都优先用桥本身的星数。

## R（收益）

- 稳定性高：域对象（Cell、BuildingState）不直接依赖渲染逻辑，规则变更不会破坏预制体适配
- AI 可用：纯 C# 状态（BuildingState）可被 AIBoardState 直接拷贝，无需转换
- 编辑器友好：Cell 使用 ExecuteAlways，在 Edit Mode 也能正确构建预制体层级
- 轴心一致性：FitInstanceToCell 统一处理预制体到格子的缩放/旋转/位置适配，渲染层零特殊处理
