# 组件化单位架构

## S（场景）

战棋游戏中单位能力差异极大：步兵能移动、能战斗、能占领建筑；坦克能移动能战斗但不能占领；补给车能移动能补给但不能战斗。能力组合如果写在一个大单体类里，耦合严重，难以按需配置。

## T（挑战）

如何让不同单位按需组合能力，且各能力之间不直接耦合？

## A（方案）

采用组合模式 + 接口隔离。每个能力独立为 MonoBehaviour 组件：

| 组件 | 职责 | 暴露接口 |
|------|------|----------|
| `UnitHealth` | HP 管理、死亡处理 | — |
| `UnitMovement` | 移动执行、路径动画 | — |
| `UnitCombat` | 战斗执行、伤害计算、特效 | — |
| `UnitCapture` | 占领能力 | `ICapturable` |
| `UnitSupply` | 补给能力 | `ISupplier` |
| `UnitTransport` | 装载/卸载 | `ITransporter` |

`UnitController` 作为门面，只持有各组件引用，不实现任何具体能力。各组件通过 static 事件向外广播（如 `OnDeath`、`OnAttackStarted`），表现层订阅这些事件而非直接调用组件。

能力组合通过在 Prefab 上挂载不同组件实现：步兵挂载 Capture，运输载具挂载 Transport，补给车挂载 Supply。

## R（收益）

- 配置灵活：Inspector 里增减组件即可改变单位能力，无需改代码
- 职责清晰：每个组件只管一件事，测试和 Debug 容易定位
- AI 模拟无障碍：AI 使用纯数据 `AIUnitSnapshot`，与组件完全解耦
- 事件驱动：组件之间无直接引用，通过事件总线通信，扩展新能力不破坏现有代码
