# 只读视图接口体系

## S（场景）

战棋游戏中，规则逻辑（"这个单位能不能攻击那个目标"）需要对两种完全不同的数据源求值：一是运行时 Unity 场景中的真实 GameObject，二是 AI 搜索中每帧可能 Clone 数十次的纯数据快照。如果规则直接写死在具体类里，运行时和 AI 就得各写一份——代码重复，分叉风险极高。

## T（挑战）

如何让同一套规则代码既能在 Unity 运行时生效，又能在 AI 模拟层运行，而不需要维护两份实现？

## A（方案）

抽象出四个只读视图接口，将"读取世界状态"标准化：

```csharp
public interface ICellReadView { int TerrainStars { get; } bool HasUnit { get; } bool HasBuilding { get; } }
public interface IUnitReadView { bool Alive { get; } UnitFaction Faction { get; } Vector2Int GridCoord { get; } int Hp { get; } MovementType MovementType { get; } }
public interface IBuildingReadView { Vector2Int GridCoord { get; } UnitFaction OwnerFaction { get; } bool IsHq { get; } int CaptureHp { get; } }
public interface IGridReadView { int Width { get; } int Height { get; } bool IsInBounds(Vector2Int coord); bool TryGetCell(Vector2Int coord, out ICellReadView cell); }
```

运行时通过 `UnitController` / `Cell` 等 GameObject 实现这些接口；AI 侧通过 `AIBoardState` + `AICellSnapshot` / `AIUnitSnapshot` 等 struct 实现。`BoardSnapshotFactory` 负责将运行时状态拍成纯数据快照，**规则层只依赖接口**，不感知实现者是谁。

AI 搜索中的伤害计算也是典型例子：`CombatRulesShared.CalculateSnapshotDamage()` 查 `AIBoardState.damageMatrix`，和运行时 `UnitCombat.Attack()` 调用的是同一份规则定义。

## R（收益）

- **零重复**：规则只写一次，运行时和 AI 共用
- **零 GC 设计**：AI 快照用 struct + Array.Copy，Clone 高效无分配
- **接口隔离**：规则层不依赖任何 MonoBehaviour，AI 模拟彻底脱离 Unity 场景
- **易于扩展**：新增能力只需在对应 ReadView 接口加 property，无需改动规则实现者
